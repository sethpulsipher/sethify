- The body paragraph sits beside or below the stats. Regular body text. No bullet points.

## Section 3: What You Get

**Layout:** Grid of 6 feature cards. Two rows of three on desktop, single column on mobile. Each card has a heading and 1-2 sentences. No icons unless they genuinely add clarity — most do not.

**Heading:** What Your Website Includes

1. **Hand-Coded From Scratch**
No WordPress. No Squarespace. No templates from a marketplace. Your site is built line by line for speed, control, and search rankings.

- No decorative icons. If you use icons, they must be functional (like a phone icon next to a phone number). Decorative icons on feature cards are a crutch for weak copy.

## Section 4: Portfolio / Proof

**Layout:** This section is a visual break. Full-width or near full-width. Show 2-3 project examples in a grid or staggered layout.

**Heading:** Work That Speaks for Itself

**Content:** This section uses placeholder content until real case studies exist. Structure it so real projects drop in later without a redesign.

**Placeholder structure per project card:**
- Project image (large, high quality)
- Project type label (e.g., "Kitchen Remodel" or "New Construction")
- One sentence result line (e.g., "Generated 40+ quote requests in the first 3 months")

**Design notes:**
- This is the most visual section on the page. Let images breathe. Large cards with generous spacing.
- Do not use a lightbox or modal. Each project card should link to a future case study page or simply exist as a visual proof block.
- Staggered or asymmetric layout works well here to break the grid rhythm of Section 3.
- If no real images exist yet, use solid colored placeholder blocks with the project type label overlaid. Do not use stock photos.

# Design System Notes for All Industry Pages

**Typography:** DM Serif Display for H1 and section headings. DM Sans for body text, subheadlines, and UI elements.

**Colors:**
- Background: Cream (#FDF8F3) for main sections
- Text: Ink (#0F172A)
- CTA buttons: Coral (#FF6B4A) with cream or white text
- Dark sections (Final CTA): Ink background, cream text, coral button
- Cards: White or light warm gray (#F5F0EB or similar), 2px border-radius

**Spacing:** Generous. Each section should have significant vertical padding (80-120px desktop, 48-64px mobile). Do not crowd sections together.

**Border radius:** 2px. Not rounded. Not sharp. Subtle.

**Grain texture:** Applied via CSS overlay on hero and dark sections. Not baked into images.

**Icons:** Lucide React. Outline style. 24px. Use sparingly. Do not put decorative icons on every card or feature block.

**Images:** Use real project photos when available. Use solid colored placeholder blocks when not. Never use stock photos of contractors, handshakes, or people pointing at laptops.

**Mobile:** Every section stacks to single column. No horizontal scrolling. No carousels. Touch targets 44px minimum.

**Shared components between web design and AI pages:**
- Hero layout (same structure, different content)
- Stat block component
- FAQ accordion
- Final CTA dark section
- Cross-link callout banner

**Components that differ between web design and AI pages:**
- Web design uses a 6-card feature grid (Section 3)
- AI uses stacked alternating feature blocks (Section 3)
- Web design has a portfolio/proof section (Section 4)
- AI has a "The Math" ROI section (Section 5)
- Web design has a two-card pricing section (Section 6)
- AI skips pricing for now and leads with the ROI argument instead
